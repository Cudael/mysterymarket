# Category & Subcategory Taxonomy

MysteryMarket uses a two-level taxonomy: 16 top-level **categories**, each with 12–16 **subcategories** (~210 total).

## Maturity Levels

Every idea can be tagged with a maturity level to signal how developed it is:

| Value | Label | Description |
|-------|-------|-------------|
| `SEED` | Seed | A raw spark — one-liner or short paragraph |
| `CONCEPT` | Concept | Fleshed out with context, problem, and high-level solution |
| `BLUEPRINT` | Blueprint | Detailed execution plan, market analysis, or technical spec |
| `PROTOTYPE_READY` | Prototype-Ready | Includes wireframes, mockups, pseudocode, or working models |

## Adding New Categories or Subcategories

1. **Add to `CATEGORY_DATA`** in `prisma/seed-categories.ts` — this is the source of truth for seeding.
2. **Add to `CATEGORIES` array** in `src/lib/constants.ts`.
3. **Add to `CATEGORY_META`** in `src/lib/constants.ts` with `name`, `slug`, `description`, and `icon`.
4. **Add to `SUBCATEGORIES`** in `src/lib/constants.ts`.
5. **Add the icon** to `CATEGORY_ICON_MAP` in `src/lib/categories.ts` if it's a new icon.
6. **Create a migration** with the appropriate SQL changes if modifying the database schema.
7. **Re-run the seed** with `npx prisma db seed` to populate the new entries.

## Migration Strategy (Old → New)

The old flat `category String?` field on `Idea` is **preserved** for backward compatibility. New fields `categoryId` and `subcategoryId` are nullable foreign keys pointing to the `Category` and `Subcategory` tables.

### Old category mappings
| Old value | New category |
|-----------|-------------|
| `Technology` | `Software & Technology` |
| `Business` | `Startup & Business Ideas` |
| `Marketing` | `Marketing & Growth` |
| `Design` | `Design & Visual Arts` |
| `Education` | `Education & Knowledge` |
| `Health` | `Health & Wellness` |
| `Finance` | `Finance & Monetization` |
| `Entertainment` | `Content & Creator Economy` |
| `Science` | `Experimental & Wildcard` |
| `Other` | `Experimental & Wildcard` |
| `Creative` | `Content & Creator Economy` |

To migrate existing ideas, run a one-time data migration that sets `category` to the new name and optionally resolves `categoryId` by looking up the `Category` table by slug.
